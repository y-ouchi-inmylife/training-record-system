## 2. 導入の背景と目的

### 2.1 現在の課題

現在、相談機関では以下のような課題を抱えています。

- **複数のツールでバラバラに管理**
  - Excel、Access、Notesなど、機関ごとに異なるツールで管理しており、統一されていない
  
- **クライアント情報の項目が多すぎて管理が困難**
  - 80項目近い情報を管理する必要があるが、Excelでは1画面で見切れず、横スクロールが必要
  - 情報の入力・確認に時間がかかる
  
- **検索性の低さ**
  - 過去の相談記録を検索するのに時間がかかる
  - クライアントの支援経過を時系列で把握しづらい
  
- **データの共有が非効率**
  - 複数のカウンセラーで情報を共有する際に、ファイルの受け渡しが発生
  - 最新版がどれかわからなくなる

### 2.2 なぜシステム化するのか

上記の課題を解決し、カウンセラーが本来の支援業務に集中できる環境を整えるため、専用の相談記録管理システムを導入します。

### 2.3 導入後に実現したいこと

- **効率的な情報管理**
  - クライアント情報を見やすく整理し、必要な情報にすぐアクセスできる
  - セクション分けされた画面で、段階的に情報を入力・更新できる
  
- **迅速な検索・参照**
  - クライアント名、最終相談日などで素早く検索できる
  - 過去の相談記録を時系列で一覧表示できる
  
- **チーム内での情報共有**
  - 複数のカウンセラーが同じシステムで情報を共有できる
  - 最新の情報が常に反映される
  
- **支援の質の向上**
  - クライアントの支援経過を可視化し、適切な支援を提供できる
  - 相談記録とカウンセラー所感を分けて管理し、チーム内での情報共有を促進

## 3. 現状整理（As-Is）

### 3.1 現在の業務フロー

**クライアント初回来談時**
1. 紙の相談申込書に記入してもらう
2. カウンセラーがExcel/Access/Notesに手入力
3. クライアント情報（氏名、連絡先、学歴、職歴、障害者手帳など）を登録
4. 初回相談を実施
5. 相談記録をExcel/Access/Notesに記録

**2回目以降の相談時**
1. 事前にクライアント情報を確認（Excelファイルを開いて検索）
2. 過去の相談記録を確認（スクロールして該当箇所を探す）
3. 相談を実施
4. 相談記録を追記

**カウンセラー間での情報共有**
1. 担当変更や引き継ぎ時に、ファイルを共有フォルダにコピー
2. 複数人が同時に編集すると、ファイルが競合する
3. どのファイルが最新版かわからなくなることがある

### 3.2 現在の問題点

| 問題点 | 詳細 |
|---|---|
| **画面の見づらさ** | クライアント情報の項目が多く、Excelでは横スクロールが必要。一覧性が低い。 |
| **検索の困難さ** | 「最終相談日が6ヶ月以上前のクライアント」などの条件検索ができない。 |
| **入力の手間** | 同じ情報を何度も入力する必要がある（プルダウンがない）。 |
| **ファイル管理の煩雑さ** | ファイル名に日付を入れてバージョン管理しているが、混乱しやすい。 |
| **同時編集の制約** | 複数人が同時に編集できない（Accessは可能だが、設定が複雑）。 |

### 3.3 手作業箇所

- クライアント情報の手入力
- 相談記録の手入力
- 過去の記録の検索（目視でスクロール）
- バックアップ（手動でファイルをコピー）
- データの集計（必要に応じてExcelで手作業）

## 4. 将来像（To-Be）

### 4.1 システム導入後の業務イメージ

**クライアント初回来談時**
1. 初回専用入力画面を開く
   - カウンセラーが聞き取りながら入力、または
   - クライアント自身がタブレット等で入力
2. クライアント情報を登録（全カテゴリー入力可能、必須項目のみでもOK）
3. 初回相談を実施
4. 相談記録登録画面で記録を入力（インテークフラグをチェック）

**2回目以降の相談時**
1. システムにログイン
2. クライアント名で検索し、詳細画面を表示
3. 過去の相談記録を時系列で一覧確認
4. 相談を実施
5. 相談記録を登録（クライアントに紐づけて自動保存）

**カウンセラー間での情報共有**
1. 全てのカウンセラーが同じシステムにアクセス
2. 最新の情報が常に反映されている
3. ファイルの受け渡しやバージョン管理が不要

### 4.2 改善点

| 改善項目 | 改善内容 |
|---|---|
| **画面の見やすさ** | クライアント情報をカテゴリーに分類。セクションごとに折りたたみ可能で、必要な情報だけを表示。 |
| **検索機能の強化** | 名前（部分一致）、最終相談日などで素早く検索可能。 |
| **入力の効率化** | プルダウン選択による入力ミス防止。マスタデータは管理者が編集可能。 |
| **データ管理の一元化** | LAN内の複数PCから同じデータベースにアクセス可能。 |
| **自動バックアップ** | 毎日自動でバックアップファイルを作成。データ消失リスクを軽減。 |

### 4.3 期待される効果

- **業務時間の削減**
  - クライアント情報の検索時間が大幅に短縮
  - 相談記録の入力がスムーズになる
  
- **支援の質の向上**
  - クライアントの支援経過を時系列で把握しやすくなる
  - カウンセラー所感を分けて管理することで、チーム内での情報共有が促進される
  
- **データの安全性向上**
  - 自動バックアップによりデータ消失リスクを軽減
  - ローカル環境で運用するため、外部への情報漏洩リスクがない
  
- **将来的な拡張性**
  - プログラム管理機能や統計レポート機能など、将来的な機能追加が容易

## 5. システム概要

### 5.1 システムの全体像

本システムは、相談機関におけるクライアント情報と相談記録を一元管理するためのWebアプリケーションです。
ブラウザからアクセスし、直感的な操作で情報の登録・検索・閲覧が可能です。

### 5.2 利用環境

**推奨環境**
- **OS**：Windows 11以上
- **ブラウザ**：Google Chrome、Microsoft Edge（最新版）
- **ネットワーク**：
  - 単独PC利用：ネットワーク不要
  - 複数PC利用：LAN内ネットワーク

**動作イメージ**

【単独PC利用の場合】
```
PC（Windows）
├─ PostgreSQLデータベース（ローカルインストール）
├─ アプリケーションサーバー（ローカルで起動）
└─ ブラウザでアクセス（http://localhost:8000）
```

【LAN内複数PC利用の場合】
```
サーバー役PC（IPアドレス：192.168.1.100）
├─ PostgreSQLデータベース
└─ アプリケーションサーバー

クライアントPC1 ───┐
クライアントPC2 ───┼─ LAN経由でアクセス（http://192.168.1.100:8000）
クライアントPC3 ───┘
```

### 5.3 利用人数想定

- **同時利用カウンセラー数**：最大10名
- **管理クライアント数**：
  - 支援中：約100名
  - 累計（過去含む）：約1,000名

### 5.4 技術構成

本システムは以下の技術で構築されます。

| 要素 | 技術 | 説明 |
|---|---|---|
| **データベース** | PostgreSQL | クライアント情報、相談記録を保存 |
| **サーバー側** | Laravel（PHPフレームワーク） | データ処理、画面表示を制御 |
| **画面表示** | Blade / Vue.js | ブラウザで表示される画面 |
| **動作環境** | Windows ローカル環境 | インターネット接続不要（運用時） |

※技術的な詳細は開発者が管理し、利用者（カウンセラー）は通常のWebサイトと同様にブラウザで操作します。

### 5.5 データの保存場所

- **すべてのデータはローカルPC/サーバー内に保存**されます
- **外部サーバーへの送信は一切ありません**
- インターネット接続は不要です（インストール時のみ必要）

## 6. 利用者と権限

本システムでは、2つの権限レベルを設定します。

### 6.1 権限一覧

| 利用者 | できること |
|---|---|
| **管理カウンセラー** | ・カウンセラーアカウントの追加・編集・削除<br>・マスタデータの編集（相談内容、フェーズなど）<br>・全クライアント情報の閲覧・編集・削除<br>・全相談記録の閲覧・編集・削除<br>・データのインポート/エクスポート<br>・バックアップの実行 |
| **一般カウンセラー** | ・全クライアント情報の閲覧・編集・削除<br>・全相談記録の閲覧・編集・登録<br>・データのエクスポート |

### 6.2 権限設計の考え方

**全てのカウンセラーが全クライアント情報を閲覧可能**
- 担当カウンセラーの不在時にも対応できるよう、情報共有を重視
- チーム全体でクライアントを支援する体制

**管理カウンセラーのみがアカウント管理とマスタ管理を実施**
- 不要なアカウントの追加を防止
- マスタデータの整合性を保つ

### 6.3 初期アカウント

システム導入時に、以下のアカウントを作成します。

- **管理カウンセラーアカウント**：1名
- **一般カウンセラーアカウント**：必要数（導入時に決定）

※アカウントの追加は、管理カウンセラーが管理画面から実施できます。

## 7. 機能一覧

本システムで提供する機能を以下に示します。

### 7-1. クライアント情報管理

#### 7-1-1. 初回専用入力画面
- **概要**：初回来談時に使用する入力画面。カウンセラーまたはクライアント自身が使用。
- **入力項目**：クライアント情報の全カテゴリー（段階的入力可能）
- **特徴**：
  - シンプルで分かりやすいUI
  - 必須項目のみの簡易入力も可能

#### 7-1-2. クライアント新規登録
- **概要**：新規クライアントの情報を登録する
- **入力項目**：50項目（7カテゴリー）※詳細は「8. データ項目定義」参照
- **条件**：
  - 必須項目は検討中
  - その他の項目は任意（段階的に入力可能）
- **出力**：登録完了後、クライアント詳細画面に遷移

#### 7-1-3. クライアント情報編集
- **概要**：既存クライアントの情報を編集する
- **入力項目**：50項目（7カテゴリー）
- **条件**：
  - カテゴリーごとに編集可能
  - 編集中は他のカウンセラーに編集中であることを通知（推奨）
- **出力**：更新完了後、クライアント詳細画面に戻る

#### 7-1-4. クライアント情報削除
- **概要**：クライアント情報を削除する
- **条件**：
  - 管理カウンセラーのみ実行可能
  - 削除確認メッセージを表示
  - 紐づく相談記録も同時に削除される旨を警告
- **出力**：削除完了後、クライアント一覧画面に戻る

#### 7-1-5. クライアント詳細表示
- **概要**：クライアントの詳細情報を表示する
- **表示内容**：
  - 基本情報（7カテゴリー）をセクション別に表示
  - 各セクションは折りたたみ可能
  - 相談記録一覧（時系列、新しい順）
  - 参加したプログラム一覧（将来拡張）
- **特徴**：
  - 未入力セクションには「未入力」と表示
  - 最終更新日を各セクションに表示

#### 7-1-6. クライアント検索
- **概要**：条件を指定してクライアントを検索する
- **検索条件**：
  - 名前（部分一致）※必須
  - 最終相談日（期間指定）
  - 支援状態（プルダウン）
  - 主担当カウンセラー（プルダウン）
- **出力**：検索結果一覧を表示

---

### 7-2. 相談記録管理

#### 7-2-1. 相談記録新規登録
- **概要**：相談後に記録を登録する
- **入力項目**：15項目 ※詳細は「8. データ項目定義」参照
  - 相談日、相談時間、クライアント選択
  - インテーク/フォローアップフラグ
  - 相談内容（プルダウン）、相談内容詳細
  - 担当者1、担当者2
  - 相談記録、所感
  - フェーズ、出欠状況、相談形態、相談形態詳細
- **相談参加者の登録**：
  - 参加者を複数追加可能
  - 参加者区分（本人/家族/支援者/その他）と詳細を入力
- **条件**：
  - 必須項目：相談日、クライアント、相談内容、担当者1
- **出力**：登録完了後、クライアント詳細画面（相談履歴）に戻る

#### 7-2-2. 相談記録編集
- **概要**：既存の相談記録を編集する
- **入力項目**：15項目
  - 相談日、相談時間、クライアント選択
  - インテーク/フォローアップフラグ
  - 相談内容（プルダウン）、相談内容詳細
  - 担当者1、担当者2
  - 相談記録、所感
  - フェーズ、出欠状況、相談形態、相談形態詳細
- **相談参加者の編集**：
  - 参加者を追加・削除可能
- **条件**：編集履歴を記録（推奨）
- **出力**：更新完了後、クライアント詳細画面に戻る

#### 7-2-3. 相談記録削除
- **概要**：相談記録を削除する
- **条件**：
  - 管理カウンセラーのみ実行可能
  - 削除確認メッセージを表示
- **出力**：削除完了後、クライアント詳細画面に戻る

#### 7-2-4. 相談履歴表示（時系列）
- **概要**：クライアントの相談履歴を時系列で表示する
- **表示内容**：
  - 相談日、相談内容、担当者、フェーズ、出欠状況
  - 新しい順に表示
- **機能**：
  - 相談記録をクリックで詳細表示
  - フェーズの変遷が一目でわかる

#### 7-2-5. 相談記録検索
- **概要**：条件を指定して相談記録を検索する
- **検索条件**：
  - 相談日（期間指定）
  - 相談内容（プルダウン）
  - 担当カウンセラー（プルダウン）
  - キーワード（相談記録・所感内の全文検索）
- **出力**：検索結果一覧を表示

---

### 7-3. マスタ管理

#### 7-3-1. 相談内容マスタ管理
- **概要**：相談内容のプルダウン選択肢を管理する
- **権限**：管理カウンセラーのみ
- **機能**：
  - 選択肢の追加・編集・削除
  - 並び順の変更
- **デフォルト値**：就労相談、生活相談、心理相談、家族相談、その他

#### 7-3-2. フェーズマスタ管理
- **概要**：フェーズのプルダウン選択肢を管理する
- **権限**：管理カウンセラーのみ
- **機能**：
  - 選択肢の追加・編集・削除
  - 並び順の変更
- **デフォルト値**：
  - 【個人支援】本人と関わっていない
  - 【個人支援】相談員との関わりが中心
  - 【集団支援】他の利用者との関わりが中心
  - 【集団支援】プログラムに参加している
  - 【集団支援】社会との関わりが中心
  - 【就職支援】社会との関わりが中心
  - 【就職支援】就職活動をしている
  - 【就職支援】就労している

---

### 7-4. 認証・アカウント管理

#### 7-4-1. ログイン機能
- **概要**：カウンセラーがシステムにログインする
- **入力項目**：
  - ユーザーID
  - パスワード
- **条件**：
  - 5回連続ログイン失敗でアカウントロック（推奨）
- **出力**：ログイン成功後、ダッシュボード画面に遷移

#### 7-4-2. ログアウト機能
- **概要**：システムからログアウトする
- **出力**：ログイン画面に戻る

#### 7-4-3. カウンセラーアカウント管理
- **概要**：カウンセラーのアカウントを管理する
- **権限**：管理カウンセラーのみ
- **機能**：
  - アカウントの追加（氏名、メールアドレス、初期パスワード）
  - アカウントの編集（氏名、権限変更）
  - アカウントの削除
  - パスワードリセット
- **条件**：
  - 管理カウンセラーは最低1名必要（削除不可）

---

### 7-5. データ管理

#### 7-5-1. CSVインポート機能
- **概要**：既存データをCSV形式で一括登録する
- **権限**：管理カウンセラーのみ
- **対象データ**：
  - クライアント情報
- **条件**：
  - CSVフォーマットは指定形式に従う
  - エラー行がある場合は登録せず、エラー内容を表示
- **出力**：インポート完了件数とエラー内容を表示

#### 7-5-2. CSVエクスポート機能
- **概要**：データをCSV形式で出力する
- **権限**：全カウンセラー
- **対象データ**：
  - クライアント情報一覧
  - 相談記録一覧
- **条件**：
  - 検索・フィルタ結果をそのままエクスポート可能
- **出力**：CSVファイルをダウンロード

#### 7-5-3. 自動バックアップ機能
- **概要**：データベース全体を定期的にバックアップする
- **実行タイミング**：毎日深夜（時刻は設定可能）
- **保存先**：指定フォルダ（外付けHDD、NASなど）
- **保存世代数**：7世代（7日分）
- **機能**：
  - 手動バックアップも実行可能（管理カウンセラーのみ）
  - バックアップファイル名に日付を付与

#### 7-5-4. バックアップからの復元機能
- **概要**：バックアップファイルからデータを復元する
- **権限**：管理カウンセラーのみ
- **条件**：
  - 復元確認メッセージを表示（現在のデータが上書きされる旨を警告）
- **出力**：復元完了後、ログイン画面に戻る

### 7-6. システム設定

#### 7-6-1. システム設定管理
- **概要**：システム全体の設定を管理する
- **権限**：管理カウンセラーのみ
- **設定項目**：
  - 自動バックアップ時刻の設定
  - バックアップ保存先フォルダの設定
  - 自動ログアウト時間の設定
- **条件**：
  - 設定変更後、即座に反映される
  - 設定内容はデータベースに保存される
- **出力**：設定保存完了メッセージ

## 10. データの取り扱い

### 10.1 保存する情報

本システムでは、以下の情報を管理します。

- **クライアント情報**：氏名、連絡先、学歴、職歴、障害・医療情報、生活状況、支援管理情報など（50項目）
- **相談記録**：相談日時、相談内容、相談記録、所感、参加者情報など（15項目）
- **マスタデータ**：相談内容、フェーズの選択肢
- **カウンセラーアカウント情報**：氏名、ユーザーID、パスワード（暗号化）、権限

### 10.2 データの所有権

- **データの所有権は各相談機関に帰属します**
- システム開発事業者は、導入・保守時を除き、データにアクセスしません
- 各機関が独立したデータベースを持ち、他機関とデータを共有することはありません

### 10.3 データの保存場所

- **すべてのデータは相談機関のPC/サーバー内にローカル保存されます**
- **外部サーバーへの送信は一切ありません**
- **インターネット接続は不要です**（運用時）

### 10.4 バックアップ方法

#### 10.4.1 自動バックアップ
- **実行タイミング**：
  - 毎日指定時刻（デフォルト：午前3時）
  - PCが起動していない場合は、次回起動時に自動実行
- **保存先**：指定フォルダ（外付けHDD、NASなどを推奨）
- **保存世代数**：7世代（7日分）
- **ファイル名**：`backup_YYYYMMDD_HHMMSS.sql`

#### 10.4.2 手動バックアップ
- 管理カウンセラーがいつでも実行可能
- 重要な作業の前（大量データ削除、システム更新など）に推奨

### 10.5 データ保存期間

- **システム上の保存期間制限はありません**
- 各相談機関の規定に従い、不要なデータは定期的に削除することを推奨します
- クライアント情報削除時は、紐づく相談記録も同時に削除されます

### 10.6 データ移行

#### 10.6.1 既存データの取り込み
- 既存のExcel/Access/Notesデータは、CSV形式に整形した上でインポート可能
- CSVフォーマットは別途提供します
- データ整形・インポート作業は各機関で実施していただきます

#### 10.6.2 PC買い替え時のデータ移行
1. 旧PCでバックアップファイルを作成
2. バックアップファイルを外付けHDD/USBメモリに保存
3. 新PCにシステムをインストール
4. バックアップファイルから復元

※導入・移行時はシステム開発事業者がリモートまたは現地でサポートします

### 10.7 個人情報の取り扱い

本システムで管理する情報には、以下の個人情報が含まれます。

- **要配慮個人情報**：
  - 障害者手帳情報
  - 通院先・服薬情報
  - 相談記録・所感
  
- **一般個人情報**：
  - 氏名、生年月日、性別
  - 連絡先（電話番号、メールアドレス、住所）
  - 学歴・職歴

**取り扱い上の注意**：
- ローカル環境で運用するため、外部への情報漏洩リスクは低い
- PCの画面ロック、退席時のログアウトを徹底
- バックアップファイルの保管場所に注意（鍵付きキャビネット推奨）
- 不要になったバックアップファイルは確実に削除

### 10.8 データのエクスポート

- クライアント情報、相談記録をCSV形式でエクスポート可能
- エクスポートしたファイルは、統計分析や外部機関への報告資料作成に利用可能
- エクスポートファイルの取り扱いには十分注意してください

## 11. セキュリティ・運用

### 11.1 パスワード管理

#### 11.1.1 パスワードポリシー
- **最小文字数**：8文字以上
- **推奨**：英大文字・小文字・数字・記号を組み合わせる
- **初期パスワード**：管理カウンセラーが設定し、初回ログイン時に変更を促す

#### 11.1.2 パスワードの保存
- データベース内ではパスワードを暗号化して保存
- システム開発事業者を含め、誰もパスワードの平文を閲覧できません

#### 11.1.3 パスワード忘れ時の対応
- 管理カウンセラーがパスワードリセットを実行
- リセット後、一時パスワードを発行し、初回ログイン時に変更

### 11.2 アカウントロック

- **連続ログイン失敗**：5回連続で失敗するとアカウントをロック（推奨）
- **ロック解除**：管理カウンセラーがアカウント管理画面から解除

### 11.3 データ保存場所とアクセス制限

- **データ保存場所**：相談機関のPC/サーバー内（ローカル環境）
- **外部アクセス**：インターネット経由での外部からのアクセスは不可
- **LAN内アクセス**：LAN内の許可されたPCからのみアクセス可能

### 11.4 セッション管理

- **自動ログアウト**：一定時間操作がない場合、自動的にログアウト（推奨：15分）
- **手動ログアウト**：退席時は必ずログアウトを実施

### 11.5 障害時の対応範囲

#### 11.5.1 システム開発事業者の対応範囲
- システムの不具合修正
- バックアップからの復元サポート
- PC買い替え時のデータ移行サポート
- 操作方法の問い合わせ対応

#### 11.5.2 各機関での対応事項
- 日常的なバックアップ確認
- PCやネットワークのハードウェア障害対応
- Windows OSのアップデート
- セキュリティソフトの導入・更新

### 11.6 運用上の推奨事項

#### 11.6.1 日常運用
- **毎日の確認**：バックアップが正常に実行されているか確認
- **退席時**：必ずログアウト、または画面ロック
- **定期的な確認**：ディスク容量、バックアップファイルの保存状況

#### 11.6.2 定期メンテナンス
- **月1回**：バックアップからの復元テスト（推奨）
- **年1回**：不要なクライアント情報の整理・削除
- **システム更新時**：事前にバックアップを取得

#### 11.6.3 セキュリティ対策
- PCに画面ロック機能を設定
- ウイルス対策ソフトを導入・更新
- バックアップファイルは鍵付きキャビネットまたは物理的に安全な場所に保管
- 不要になったバックアップファイルは確実に削除

### 11.7 障害発生時の連絡先

システムに関する不具合や操作上の問題が発生した場合は、以下に連絡してください。

- **連絡先**：［システム開発事業者の連絡先を記載］
- **対応時間**：［営業時間を記載］
- **緊急時対応**：［緊急連絡先があれば記載］

以下の条件を満たした場合、システムの検収完了とします。

#### 15.2.1 機能面
- ✅ 要件定義書に記載された全機能が実装されていること
- ✅ 受入テストで致命的な不具合が発見されていないこと
- ✅ 発見された軽微な不具合が修正されていること

#### 15.2.2 性能面
- ✅ クライアント100件程度のデータ検索が3秒以内に完了すること
- ✅ 画面遷移が5秒以内に完了すること

#### 15.2.3 ドキュメント面
- ✅ 操作マニュアルが提供されていること
- ✅ CSVインポート用フォーマットが提供されていること
- ✅ システム設定手順が記載されていること

### 15.3 不具合の分類と対応

受入テスト期間中に発見された不具合は、以下のように分類し対応します。

#### 15.3.1 致命的不具合（Critical）
- **定義**：システムが起動しない、データが消失する、セキュリティ上の重大な問題など
- **対応**：即座に修正。検収は修正完了後に延期

#### 15.3.2 重大な不具合（Major）
- **定義**：主要機能が動作しない、データ整合性に問題がある、など
- **対応**：受入テスト期間中に修正

#### 15.3.3 軽微な不具合（Minor）
- **定義**：表示の乱れ、使いにくい操作、誤字脱字など
- **対応**：受入テスト期間中に修正。または、運用開始後に修正

#### 15.3.4 改善要望（Enhancement）
- **定義**：要件定義書にない機能追加、デザイン変更など
- **対応**：別途見積もり。または、将来のバージョンアップで対応

### 15.4 検収の確認者

- **確認者**：導入機関の責任者（管理カウンセラー）
- **確認方法**：受入テスト結果報告書を提出し、承認を得る
- **検収完了の証明**：検収完了報告書に署名・押印

### 15.5 受入後の修正対応

#### 15.5.1 保証期間
- **期間**：本番稼働後1ヶ月間
- **対応範囲**：受入テストで発見できなかった不具合の修正（無償）

#### 15.5.2 保証期間後
- **軽微な不具合修正**：無償対応（ケースバイケース）
- **機能追加・仕様変更**：別途見積もり

## 16. 変更管理ルール

### 16.1 変更管理の目的

開発途中での仕様変更は、以下の影響を及ぼす可能性があります。

- 開発スケジュールの遅延
- 開発コストの増加
- システム品質への影響

これらのリスクを最小限に抑えるため、変更管理ルールを定めます。

### 16.2 変更要求の手続き

#### 16.2.1 変更要求の提出
- **提出方法**：書面（メール可）で変更内容を記載
- **記載内容**：
  - 変更箇所（画面名、機能名）
  - 変更内容（現状 → 変更後）
  - 変更理由
  - 希望納期

#### 16.2.2 影響調査
システム開発事業者は、変更要求を受け取り次第、以下を調査します。

- 影響範囲の特定（関連する機能、データ）
- 作業工数の見積もり
- スケジュールへの影響
- 追加費用の有無

#### 16.2.3 見積もり提示
- 調査結果を書面で提示（3営業日以内）
- 変更内容、作業工数、追加費用、納期への影響を明記

#### 16.2.4 承認・実施
- 導入機関が見積もりを承認した場合、変更作業を実施
- 承認されない場合、変更要求は却下

### 16.3 変更の分類と対応

#### 16.3.1 軽微な変更（無償対応）
以下のような変更は、追加費用なしで対応します。

- 表示文言の修正（誤字脱字、表現の改善）
- 画面レイアウトの微調整（ボタン配置など）
- 要件定義書の記載ミスによる修正
- 受入テスト期間中の不具合修正

**条件**：作業工数が2時間以内、スケジュールに影響しないこと

#### 16.3.2 中規模な変更（有償対応）
以下のような変更は、追加費用が発生します。

- 項目の追加・削除（データベース変更を伴う）
- 画面の大幅な変更
- 新しい機能の追加
- 要件定義書にない仕様の実装

**対応**：見積もり提示 → 承認後に実施

#### 16.3.3 大規模な変更（再見積もり）
以下のような変更は、プロジェクト全体の見直しが必要です。

- システム全体の設計変更
- 技術スタックの変更
- 要件定義書の大幅な見直し

**対応**：別途協議、再契約の可能性あり

### 16.4 変更管理の記録

すべての変更要求と対応結果は、以下の形式で記録します。

| 変更No. | 要求日 | 変更内容 | 影響範囲 | 工数 | 追加費用 | 承認日 | 完了日 |
|---|---|---|---|---|---|---|---|
| CR-001 | 2026/4/10 | 検索条件に「年齢」を追加 | 検索画面 | 4時間 | 20,000円 | 2026/4/12 | 2026/4/15 |

### 16.5 変更による納期延長

- 変更作業により、当初の納期が守れない場合は、速やかに報告します
- 新しい納期を協議の上、合意します
- 納期延長による追加費用は発生しません（変更作業自体の費用は別）

### 16.6 変更の凍結期間

以下の期間は、仕様変更を原則として受け付けません。

- **結合テスト期間中**：不具合修正のみ対応
- **受入テスト期間中**：致命的不具合、重大な不具合のみ対応
- **本番稼働直前（1週間）**：緊急の不具合のみ対応

### 16.7 変更管理の責任者

- **導入機関側**：変更要求の提出者（管理カウンセラー）
- **開発側**：プロジェクトマネージャー（システム開発事業者）

## 17. 保守範囲

### 17.1 保守の定義

本システムの保守とは、以下の活動を指します。

- システムの不具合修正
- 操作方法の問い合わせ対応
- バックアップからの復元サポート
- PC買い替え時のデータ移行サポート
- 軽微な機能改善

### 17.2 保守期間

- **開始日**：本番稼働日
- **期間**：［保守期間を記載］
  - 例：本番稼働後1年間
  - 例：継続的に保守（年間契約）

### 17.3 保守対応範囲

#### 17.3.1 無償対応範囲

以下の内容は、保守期間中は追加費用なしで対応します。

**不具合修正**
- システムの動作不良の修正
- データ不整合の修正
- セキュリティ上の問題への対応

**操作サポート**
- 操作方法の問い合わせ対応（メール、電話）
- 操作マニュアルの補足説明

**データ関連**
- バックアップからの復元サポート
- PC買い替え時のデータ移行サポート
- データベースのメンテナンス（再構築、最適化）

**軽微な改善**
- 表示文言の修正
- 使いにくい操作の改善（工数2時間以内）

#### 17.3.2 有償対応範囲

以下の内容は、別途見積もりを提示し、承認後に対応します。

**機能追加**
- 新しい機能の追加
- 項目の追加・削除
- 画面の大幅な変更

**カスタマイズ**
- 特定の機関向けの独自機能
- 帳票出力機能の追加
- 統計レポート機能の追加

**データ移行**
- 大量データの一括変換・修正
- データのクリーニング作業

**拡張開発**
- プログラム管理機能の追加
- 経過の可視化機能の追加
- スケジュール・予約管理機能の追加

#### 17.3.3 保守対象外

以下の内容は、保守範囲に含まれません。

**ハードウェア・インフラ**
- PCやサーバーの故障・修理
- ネットワーク機器の故障・設定
- Windows OSの不具合
- PostgreSQLのバージョンアップ（システム開発事業者が実施する場合は有償）

**運用上の問題**
- 誤操作によるデータ削除（バックアップからの復元サポートは対応）
- 導入機関側でのデータ整形作業
- データ入力代行

**他機関への展開**
- 2機関目以降の導入
- 機関ごとのカスタマイズ

### 17.4 保守対応方法

#### 17.4.1 対応窓口
- **連絡先**：［システム開発事業者の連絡先を記載］
- **受付時間**：［営業時間を記載］
  - 例：平日 9:00〜17:00（土日祝日を除く）

#### 17.4.2 対応手段
- **リモート対応**：原則としてリモートで対応
- **現地対応**：必要に応じて現地訪問（交通費は別途請求）

#### 17.4.3 対応期限
問い合わせ内容により、以下の期限で対応します。

| 重要度 | 対応期限 | 例 |
|---|---|---|
| **緊急** | 24時間以内に初動対応 | システムが起動しない、データが消失した |
| **高** | 3営業日以内に対応 | 主要機能が動作しない |
| **中** | 1週間以内に対応 | 一部機能の不具合、操作性の問題 |
| **低** | 次回メンテナンス時に対応 | 表示の改善、軽微な使いにくさ |

### 17.5 保守の更新

- 保守期間終了の1ヶ月前に、更新の意思確認を行います
- 更新する場合は、新たな保守契約を締結します
- 保守契約を終了する場合は、以下の点にご注意ください：
  - システムの不具合が発生しても対応できません
  - 操作サポートを受けられません
  - データ移行サポートを受けられません

### 17.6 保守の記録

保守対応の履歴は、以下の形式で記録します。

| 対応No. | 受付日 | 内容 | 重要度 | 対応日 | 対応内容 | 担当者 |
