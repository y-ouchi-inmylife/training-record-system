## 4. システム概要

### 4-1. システムの全体像

本システムは、トレーニング機関におけるクライアント情報とトレーニング記録を一元管理するためのWebアプリケーションです。
ブラウザから利用でき、クライアント情報の管理、トレーニング記録の作成・参照、トレーナー間での情報共有を行えます。

また、本システムはクライアント（飼い主）向けの閲覧機能を備えています。
トレーナーが登録したトレーニング記録やメディア（写真・動画）を、クライアント自身がメールアドレスとパスワードでログインして閲覧できます。
この閲覧機能はトレーナー向けの管理機能とは独立しており、クライアントは自分に紐づく情報の閲覧のみが可能です（新規作成・編集・削除はできません）。

### 4-2. 利用環境

**推奨環境**
- **ブラウザ**：Google Chrome、Microsoft Edge（いずれも最新版）
  - 動作保証外：Firefox、Safari など
  - 非対応：Internet Explorer
- **ネットワーク**：インターネット接続必須
- **マイク**：録音機能を使用する場合はマイク（内蔵または外付け）が必要。ブラウザからマイクへのアクセス許可が必要

**※環境維持のためのお願い**
- OSとブラウザを常に最新版に保ってください。
- 安定したインターネット接続環境を維持してください。
- 各PCにウイルス対策ソフトを導入することを推奨します。

**動作イメージ**

```
サーバー
├─ MySQLデータベース
├─ Laravelアプリケーション
└─ HTTPS（SSL/TLS）

PC1 ───┐
PC2 ───┤
PC3 ───┼─ サーバーにアクセス
PC4 ───┘
```

### 4-3. 技術構成

本システムは以下の技術で構築されます。

| 要素 | 技術 | 説明 |
|---|---|---|
| データベース | MySQL 8.0 | クライアント情報、トレーニング記録などを保存 |
| サーバー側 | Laravel 12.x（PHPフレームワーク） | データ処理、画面表示を制御 |
| 画面表示 | Blade + Bootstrap 5 | ブラウザで表示される画面 |
| 外部通信 | HTTPS（SSL/TLS） | インターネット経由での安全な通信 |
| 文字起こしAPI | OpenAI Whisper API | 音声ファイルの文字起こし |
| 要約API | Claude API | 文字起こしテキストの要約 |
| 処理実行 | Laravel同期処理（sync） | 文字起こし・要約の同期実行 |

※技術的な詳細は開発事業者が管理し、利用者は通常のWebサイトと同様にブラウザで操作します。

### 4-4. 利用想定

**想定規模**
- 同時利用トレーナー数：最大10名
- 管理クライアント数：
  - 支援中：約100名
  - 累計（過去含む）：約1,000名
- トレーニング記録数：クライアント1名あたり平均200件程度（長期支援の場合は500件を超えることもある）

**同時編集の動作**
- 同一クライアント情報を複数のトレーナーが同時編集することは可能です
- 後から保存したデータが優先されます（先に保存した編集内容は上書きされる場合があります）

### 4-5. データの所有権

- データの所有権は、貴機関に帰属します。
- システム開発事業者は、貴機関からの依頼または同意のもとに限り、データにアクセスします。

## 5. 利用者と権限

### 5-1. 権限一覧

本システムでは、以下の権限レベルを設定します。

| 権限 | ①業務機能 | ②業務管理機能 | ③システム管理機能 | ④コマンド実行 | 備考 |
|---|:---:|:---:|:---:|:---:|---|
| 一般 | 〇 | × | × | × | 通常の業務操作のみ |
| 管理者 | 〇 | 〇 | × | × | 現場の管理者 |
| システム管理者 | × | × | 〇 | × | システム開発事業者が保守・緊急対応で使用。無効化・削除不可。ロック対象外（詳細は11-2-2参照） |
| システム | × | × | × | 〇 | サーバー上で実行されるコマンド処理。自動実行と、開発事業者がサーバーに直接接続（SSH）して手動実行する場合の両方を含む。Webログインによる権限ではない |

- **①業務機能**：クライアント管理やトレーニング記録の作成・閲覧などのトレーナーの日常業務。
- **②業務管理機能**：日常業務を運用するための管理業務。トレーナー管理、マスタ管理、レポートなど。
- **③システム管理機能**：システムを円滑に運用するためのシステム管理業務。定時実行バッチの設定、サーバーの容量管理など
- **④コマンド実行**：サーバー上でのコマンド実行（バッチ処理など）。

#### クライアント（閲覧利用者）

上記の権限は、システムを操作するトレーナーの区分である。これとは別に、クライアント（飼い主）は自分に紐づく情報を閲覧するための利用者として扱う。
クライアントはトレーナーの権限体系とは独立しており、メールアドレスとパスワードでログインし、自分に紐づくトレーニング記録・メディアの閲覧のみを行える。新規作成・編集・削除、および他のクライアントの情報へのアクセスはできない。
クライアントの閲覧は、トレーナーが対象クライアントの閲覧を解放した場合にのみ可能となる（閲覧解放の詳細は別途）。

#### 情報共有の方針

担当トレーナー不在時の対応、チーム全体での支援のため、すべてのトレーナーがすべてのクライアント情報・トレーニング記録（担当以外も含めて）を閲覧・編集可能。

### 5-2. 初期アカウント

システム導入時に、以下のアカウントを作成します。

- **管理者（admin）**：1名
- **システム管理者（system_admin）**：1名

**※業務で使用するトレーナーの登録について**
上記の管理者（admin）でログイン後、トレーナー管理画面から登録してください。追加完了後、管理者（admin）アカウントは削除してください。

### 6-1. 認証

#### 6-1-1. ログイン
- **概要**：システムにログインする
- **入力**：
  - ログインID
  - パスワード
- **動作**：
  - ログイン成功後、ダッシュボード画面に遷移
  - 連続してログインに失敗するとアカウントがロックされる（詳細は11-2-2参照）
- **利用権限**：認証不要（未ログイン状態）

#### 6-1-2. 強制パスワード変更
- **概要**：初回ログイン時、または管理者によるパスワードリセット直後のトレーナーに対し、パスワード変更を強制する
- **入力**：
  - 新しいパスワード（必須。詳細は11-2-1参照）
  - 新しいパスワード（確認）（必須、新しいパスワードと一致すること）
- **動作**：
  - パスワード変更が必要なトレーナーは、ログイン直後に強制パスワード変更画面へ自動遷移
  - パスワード変更完了後、ダッシュボード画面に遷移
- **利用権限**：一般、管理者、システム管理者

#### 6-1-3. ログアウト
- **概要**：システムからログアウトする
- **動作**：ログイン画面に戻る
- **利用権限**：一般、管理者、システム管理者

#### 6-1-4. セッション期限切れエラー表示
- **概要**：セッションが切れた状態でリクエストが発生した場合に、エラーページを表示する
- **条件**：
  - セッションタイムアウト後の操作
  - CSRFトークン不一致（フォーム送信時のセキュリティ検証失敗）
- **動作**：
  - エラーページを表示し、ユーザーに状況を伝える
  - 再操作のための導線（ページを再読み込み、ログイン画面に戻る）を提供
- **利用権限**：認証状態を問わず

#### 6-1-5. アクセス禁止エラー表示
- **概要**：許可されていないリソースへのアクセス時に、エラーページを表示する
- **条件**：
  - 権限のないリソースへのアクセス（管理者専用機能への一般トレーナーからのアクセスなど）
  - IPアドレス制限対象からのアクセス
- **動作**：
  - エラーページを表示し、ユーザーに状況を伝える
- **利用権限**：認証状態を問わず


### 6-2. ダッシュボード

#### 6-2-1. ダッシュボード表示
- **概要**：ログイン後の起点となる
- **機能**：
  - **よく使う機能へのリンク**：クライアント一覧、クライアント登録、トレーニング記録登録、録音、音声記録一覧、クライアント登録（URL発行）管理
  - **主担当クライアント一覧**：ログイン中のトレーナーが主担当となっているクライアントを表示
    - 表示項目：内部ID、名前、支援状態、最終トレーニング日、担当1、担当2
    - 表示対象：支援状態マスタで「ダッシュボード表示」が「はい」のもののみ
    - 並び順：最終トレーニング日の新しい順
    - 件数：最大10件
- **利用権限**：一般、管理者


### 6-3. クライアント管理

#### 6-3-1. クライアント登録
- **概要**：クライアント情報を登録する
- **入力**：
  - 「9. データ項目一覧」参照
  - 必須項目：初回日、姓  - その他の項目は任意
- **利用権限**：一般、管理者

#### 6-3-2. クライアント登録（URL発行）

- **概要**：トレーナーが発行する招待URLから、クライアントが自身の情報を入力する
- **動作**：
  - **トレーナー側**（トレーナー認証必須）：
    - **URL発行**：有効期限（1日／7日／14日／30日、デフォルト7日）・初回日・メモ等を指定して招待URLを発行する
    - **発行済みURL一覧**：発行したURLの状態（未使用・使用済み・期限切れ）を確認し、URLコピー・QRコード表示・削除を行う
  - **クライアント側**（招待URL）：
    - **事前入力**：クライアント情報のうち、支援管理を除く項目を入力する。完了後はブラウザを閉じる
      - 必須項目：姓      - その他の項目は任意
    - **エラー処理**：URLが無効・期限切れ・使用済みの場合は、エラーとなり入力不可
  - 登録後、主担当トレーナー・支援状態は未設定のため、トレーナーが後から設定する
- **備考**：
  - クライアント側は招待URLによる認証で、通常の認証は不要
  - 招待URLは推測困難なランダム文字列を含み、1回のみ使用可能
- **利用権限**：
  - トレーナー側：一般、管理者
  - クライアント側：認証不要（招待URL）

#### 6-3-3. クライアント一覧
- **概要**：クライアントを一覧表示し、条件を指定して絞り込みや並び替えができる
- **検索条件**：内部ID、名前（姓名・かな）、最終トレーニング日、支援状態、主担当トレーナー
- **並び替え**：内部ID、名前（初期は最近登録した順）
- **利用権限**：一般、管理者

#### 6-3-4. クライアント詳細
- **概要**：クライアント情報とトレーニング記録履歴を表示する
- **表示内容**：
  - 「9. データ項目一覧」参照
  - トレーニング記録一覧（トレーニング日の新しい順）
- **利用権限**：一般、管理者

#### 6-3-5. クライアント編集
- **概要**：クライアント情報を編集する
- **入力**：
  - 「9. データ項目一覧」参照
  - 必須項目：初回日、姓  - その他の項目は任意
- **利用権限**：一般、管理者

#### 6-3-6. クライアント削除
- **概要**：クライアント情報を、確認の上で削除する
- **条件**：トレーニング記録が紐づいているクライアントは削除できない
- **利用権限**：管理者

### 6-4. トレーニング記録管理

#### 6-4-1. トレーニング記録登録
- **概要**：トレーニング記録を登録する
- **入力**：
  - 「9. データ項目一覧」参照
  - 必須項目：トレーニング日、クライアント、担当1
  - その他の項目は任意
- **動作**：
  - 登録済みのメディアを、トレーニング記録に紐づける（追加・解除）。同一のメディアを複数のトレーニング記録に紐づけできる。紐づけを解除してもメディア自体は削除されない
  - 過去の音声要約をトレーニング記録欄に読み込める（読み込み後は手動編集可能）
- **利用権限**：一般、管理者

#### 6-4-2. 録音からのトレーニング記録登録
- **概要**：トレーニングの録音から、文字起こし・要約を経てトレーニング記録を登録する
- **入力**：
  - クライアント（必須、6-5-1 の録音時に選択）
- **動作**：
  - 録音完了後、追加項目（担当2）をトレーナーが入力する
  - 入力完了後、文字起こし（6-6-1）→ 要約（6-6-2）→ トレーニング記録登録（6-4-1）が順に自動実行される
  - 文字起こしまたは要約でエラーが発生した場合、後続処理は中止される
- **自動設定項目**：
  - トレーニング日：当日の日付
  - トレーニング時刻：録音開始時刻
  - クライアント：録音時に選択したクライアント
  - 担当1：録音したトレーナー
  - トレーニング記録欄：生成された要約テキスト
- **情報漏洩防止**：
  - 録音実行中はクライアント名を表示しない（画面録画等による情報漏洩防止）
- **利用権限**：一般、管理者

#### 6-4-3. トレーニング記録一覧
- **概要**：トレーニング記録を一覧表示し、条件を指定して絞り込みや並び替えができる
- **検索条件**：内部ID、名前（姓名・かな）、トレーニング日、担当トレーナー、キーワード（トレーニング記録・所感内の全文検索）
- **並び替え**：内部ID、名前、トレーニング日（初期は最近のトレーニング日順）
- **利用権限**：一般、管理者

#### 6-4-4. トレーニング記録詳細
- **概要**：トレーニング記録の情報を表示する
- **表示内容**：「9. データ項目一覧」参照
- **動作**：トレーニング記録に紐づいたメディアを一覧表示し、閲覧（写真は拡大表示、動画は再生）できる
- **利用権限**：一般、管理者

#### 6-4-5. トレーニング記録編集
- **概要**：トレーニング記録を編集する
- **入力**：
  - 「9. データ項目一覧」参照
  - 必須項目：トレーニング日、クライアント、担当1
  - その他の項目は任意
- **動作**：
  - 登録済みのメディアを、トレーニング記録に紐づける（追加・解除）。同一のメディアを複数のトレーニング記録に紐づけできる。紐づけを解除してもメディア自体は削除されない
  - 過去の音声要約をトレーニング記録欄に読み込める（読み込み後は手動編集可能）
- **利用権限**：一般、管理者

#### 6-4-6. トレーニング記録削除
- **概要**：トレーニング記録を、確認の上で削除する
- **利用権限**：管理者

### 6-5. 音声記録管理

#### 6-5-1. 音声記録登録（録音）
- **概要**：トレーニングを録音して、音声記録を登録する
- **入力**：
  - クライアント（必須）
- **自動設定項目**：
  - 表示名：録音停止時の日時（YYYYMMDD_HHMM_トレーナーのログインID 形式）
  - 登録者：ログイン中のトレーナー
  - 状態：「文字起こし待ち」
- **情報漏洩防止**：
  - 録音中はブラウザの戻るボタンと画面遷移を無効化
  - 録音完了後は自動ログアウト（クライアントが他のクライアント情報を閲覧することを防ぐため）
- **利用権限**：一般、管理者

#### 6-5-2. 音声記録登録（音声ファイルのアップロード）
- **概要**：音声ファイルをアップロードして、音声記録を登録する
- **入力**：
  - クライアント（必須）
  - 音声ファイル（必須）
    - 対応形式：mp3、m4a、wav、mp4、webm
    - 最大サイズ：500MB
- **自動設定項目**：
  - 表示名：アップロードしたファイル名（拡張子を除く）
  - 登録者：ログイン中のトレーナー
  - 状態：「文字起こし待ち」
- **利用権限**：一般、管理者

#### 6-5-3. 音声記録登録（文字起こしテキスト）
- **概要**：文字起こし済みのテキストを直接入力して、音声記録（音声ファイルなし）を登録する
- **入力**：
  - クライアント（必須）
  - 表示名（必須、デフォルト値：作成時の日時 YYYYMMDD_HHMM 形式、編集可能）
  - 文字起こしテキスト（必須）
    - iPhone等の外部録音・文字起こしアプリで生成したテキストを想定
- **自動設定項目**：
  - 登録者：ログイン中のトレーナー
  - 状態：「要約待ち」
- **利用権限**：一般、管理者

#### 6-5-4. 音声記録一覧
- **概要**：音声記録を一覧表示し、条件を指定して絞り込みができる
- **表示項目**：日時、クライアント、表示名、登録者、再生、再生時間、文字起こし、要約、状態
- **検索条件**：登録者（「全員」+ 各トレーナー、デフォルトはログイン中のトレーナー）
- **並び順**：日時の新しい順（固定）
- **機能**：
  - 再生：各行で音声の再生・一時停止・シークが可能(音声ファイルがないレコードは再生欄を空表示)
  - 詳細表示：行を選択すると、文字起こし・要約の内容を確認・編集できる（詳細は 6-5-5 参照）
- **備考**：全トレーナーが全レコードを閲覧・操作可能
- **利用権限**：一般、管理者

#### 6-5-5. 音声記録編集
- **概要**：音声記録の表示名・文字起こし・要約テキストを編集する
- **利用権限**：一般、管理者

#### 6-5-6. 音声記録削除
- **概要**：音声記録を、確認の上で削除する
- **動作**：音声記録に関連するすべてのデータ（音声ファイル（存在する場合）、文字起こしテキスト、要約テキスト）を削除する
- **利用権限**：一般、管理者

### 6-6. 文字起こし・要約

#### 6-6-1. 文字起こし
- **概要**：音声ファイルを文字起こしする
- **条件**：音声ファイルが存在する
- **動作**：
  - 状態遷移：「文字起こし待ち」→「文字起こし中」→「要約待ち」（または「エラー」）
  - 自動リトライ：最大3回試行、60秒間隔。最終試行で失敗した場合のみエラー状態に遷移
  - 再実行：文字起こし済み・要約完了後でも再実行可能。再実行時は、確認の上で既存のテキストを上書きする
- **利用権限**：一般、管理者

#### 6-6-2. 要約
- **概要**：文字起こしテキストを要約する
- **条件**：文字起こしテキストが存在する（音声ファイルの有無は問わない）
- **動作**：
  - 状態遷移：「要約待ち」→「要約中」→「要約完了」（または「エラー」）
  - 自動リトライ：最大3回試行、60秒間隔。最終試行で失敗した場合のみエラー状態に遷移
  - 再実行：要約完了後でも再実行可能。再実行時は、確認の上で既存のテキストを上書きする
- **利用権限**：一般、管理者

#### 6-6-3. 要約プロンプト設定
- **概要**：要約（6-6-2）時に使用するプロンプトを設定する
- **機能**：
  - プロンプトの編集
- **利用権限**：管理者

### 6-7. 音声ファイル容量管理

#### 6-7-1. 音声ファイル一覧
- **概要**：音声ファイルを一覧表示する
- **集計値**：総ファイル数、総容量
- **表示項目**：日時、表示名、登録者、ファイル名、使用容量
- **並び順**：日時の新しい順（固定）
- **利用権限**：システム管理者

#### 6-7-2. 音声ファイル自動削除
- **概要**：保存期間を過ぎた音声ファイルのみを自動削除し、文字起こし・要約テキストは保持する
- **設定項目**：保存期間（1〜30日、デフォルト7日）
- **動作**：
  - **自動実行**：毎日午前3時
  - 削除対象：保存期間を超過した音声ファイル（録音もしくはアップロードした日から起算）
- **利用権限**：システム管理者

#### 6-7-3. 音声ファイル手動削除
- **概要**：音声ファイルのみを手動削除し、文字起こし・要約テキストは保持する
- **利用権限**：一般、管理者

### 6-8. トレーナー管理

#### 6-8-1. トレーナー登録
- **概要**：トレーナーを登録する
- **入力**：
  - ログインID（必須）
  - 名前（必須）
  - 初期パスワード（必須。詳細は11-2-1参照）
  - 権限（必須、一般または管理者）
- **利用権限**：管理者

#### 6-8-2. トレーナー一覧
- **概要**：トレーナーを一覧表示する
- **表示内容**：ログインID、名前、権限、状態、最終ログイン、登録日
- **並び順**：表示順（6-8-4）の昇順
- **利用権限**：管理者

#### 6-8-3. トレーナー編集
- **概要**：トレーナーの名前と権限を編集する
- **条件**：
  - 自分自身は編集できない
- **利用権限**：管理者

#### 6-8-4. 表示順変更
- **概要**：トレーナー選択肢の並び順を管理する
- **動作**：設定した表示順で、以下の場面の選択肢にトレーナーが昇順表示される
  - クライアントの主担当トレーナー選択
  - トレーニング記録の担当1・担当2選択
  - 統計のトレーナー選択
- **条件**：システム管理者は対象外（選択肢に表示されない）
- **利用権限**：管理者

#### 6-8-5. パスワードリセット
- **概要**：他のトレーナーのパスワードをリセットする
- **入力**：新しいパスワード（必須。詳細は11-2-1参照）
- **動作**：リセット後、対象のトレーナーは次回ログイン時にパスワードを再設定する必要がある（6-1-2 強制パスワード変更）
- **利用権限**：管理者

#### 6-8-6. 無効化・有効化
- **概要**：トレーナーを無効化または有効化する（詳細は11-2-2参照）
- **利用権限**：管理者

#### 6-8-7. ロック解除
- **概要**：ロックがかかったトレーナーを利用可能な状態に戻す（詳細は11-2-2参照）
- **利用権限**：管理者

#### 6-8-8. トレーナー削除
- **概要**：トレーナーを、確認の上で削除する
- **条件**：以下のトレーナーは削除できない
  - クライアント情報の主担当
  - トレーニング記録の担当1、担当2
  - ログイン中の自分自身
  - 最後の有効な管理者（管理者が1名のみの場合）
  - システム管理者
- **利用権限**：管理者

#### 6-8-9. トレーナー操作履歴
- **概要**：トレーナーの操作履歴を一覧表示し、条件を指定して絞り込みができる
- **記録対象**：
  - ログイン、ログアウト
  - クライアント詳細、クライアント編集、クライアント登録、クライアント削除
  - トレーニング記録詳細、トレーニング記録編集、トレーニング記録登録、トレーニング記録削除
- **検索条件**：トレーナー、操作、開始日、終了日
- **並び順**：記録日時の新しい順（固定）
- **利用権限**：管理者

#### 6-8-10. 長期間未ログインアカウントの自動ロック
- **概要**：長期間ログインのないトレーナー（退職・休職者等）のアカウントを自動的にロックする（詳細は11-2-2参照）
- **動作**：
  - **自動実行**：毎日午前2時
- **条件**：
  - 最終ログイン日時から30日が経過したトレーナー（一度もログインしていないアカウントは対象外。6-8-11で対応）
  - システム管理者は対象外
- **利用権限**：システム

#### 6-8-11. 未使用アカウントの自動ロック
- **概要**：発行後に一度もログインされていないトレーナーのアカウントを、一定期間経過後に自動的にロックする
- **動作**：
  - **自動実行**：毎日午前2時
- **条件**：
  - 一度もログインしておらず、アカウント作成日から7日が経過したトレーナー
  - システム管理者は対象外
- **利用権限**：システム

### 6-9. マスタ管理

マスタとは、システム全体で使用する基準データ（トレーニング内容、フェーズ、支援状態などの選択肢）です。

#### 6-9-1. 支援状態

  - 並び順の変更

- **条件**：使用されている支援状態は削除不可
- **初期値**：「9. データ項目一覧」参照

  - 並び順の変更

- **初期値**：「9. データ項目一覧」参照

  - 並び順の変更

- **初期値**：「9. データ項目一覧」参照
- **利用権限**：管理者

### 6-10. セキュリティ設定

#### 6-10-1. 自動ログアウト
- **概要**：一定時間ユーザー操作がない場合、自動的にログアウトする
- **設定項目**：自動ログアウト時間（設定しない／3分／5分／10分／15分／30分／60分）
- **動作**：
  - マウス操作・キーボード入力などの画面操作があるたびにタイマーをリセット
  - 「設定しない」を選択した場合は自動ログアウトしない
- **利用権限**：管理者

#### 6-10-2. IPアドレス制限
- **概要**：指定したIPアドレスからのみシステムを利用可能にする
- **設定項目**：
  - 有効化のオン／オフ（オフにしても指定IPアドレスは保持）
  - 指定IPアドレスの追加・編集・削除（CIDR形式（例: 192.168.1.0/24）対応、各IPアドレスに備考を付与可能）
- **備考**：
  - システム管理者アカウントはIPアドレス制限の対象外
  - localhost（127.0.0.1）は常に許可
- **利用権限**：システム管理者

### 6-11. レポート

#### 6-11-1. トレーニング記録数推移
- **概要**：トレーニング記録数の推移を年度／年別、月別で集計し、性別・年齢別の内訳を表示する
- **集計条件**：
  - トレーナー選択（すべて／個別トレーナー）
  - 表示切替（年度／年、年度は4月1日〜翌年3月31日）
- **集計項目**：
  - のべトレーニング件数：対象期間のトレーニング記録の総数
  - クライアント実人数：対象期間の重複なしクライアント数
    - 性別内訳：男、女、その他、未入力
    - 年齢別内訳：～19、20～29、30～39、40～49、50～59、60～69、70～、不明
- **動作**：
  - **年度／年別推移**：データがある年度／年のみ表示
  - **月別推移**：選択した年度／年の1〜12月をすべて表示（データがない月は0で表示）
  - 表示切替（年度／年）に応じて、月別推移の対象選択も連動
- **集計仕様**：
  - トレーナー選択での集計対象は、担当1または担当2のいずれかに該当するトレーニング記録
  - 年齢・性別は初回日時点の情報で集計
  - 初回日または生年月日が不明な場合は、年齢別内訳の「不明」にカウント
- **利用権限**：管理者

## 8. バッチ一覧

**記載範囲**:
- 本章では、本システムが提供する全バッチを列挙する。
- 全バッチとは、機能一覧（6章）のうちバッチ（コマンドラインから実行する処理）として実装する機能に対応するバッチを指す。
- 「どのバッチが存在するか」の棚卸しと、各バッチの実行方式・実行権限の把握を目的とする。

**記載単位**:
- バッチは実装ファイル単位で1項目とする。1つの実装単位が複数の機能（自動実行と手動実行など）を提供することがある。

**参照ルール**:
- バッチ一覧は、各バッチの「提供機能」で機能一覧（6章）を参照する。
- 各バッチの「概要」は対応する機能の概要と整合させる（複数機能ならまとめる）。

**記載する項目**: 各バッチについて、以下の項目を記載する。

- **概要**（必須）: バッチの目的を簡潔に説明
- **提供機能**（必須）: バッチが提供する機能（6章への参照）
- **実行方式**（必須）: 自動／手動／自動と手動
- **実行権限**（必須）: バッチを実行できる権限
- **備考**: 業務的・運用的に重要な特記事項がある場合のみ記載

**記載しない項目**（委譲先は doc-index.md 参照）：

- artisan コマンド名、引数、オプション
- 実装の詳細（処理フロー、エラーハンドリング、リトライ仕様）
- cron 設定、Laravel Scheduler の記述
- ログ出力先、通知方式
- 機能の動作詳細

### 11-1. データの取り扱いに関するお願い

#### 11-1-2. 個人情報の取り扱い

本システムで管理する情報は、個人情報保護法および貴機関の個人情報保護規程に基づき適切に管理してください。

##### クライアント情報・トレーニング記録

- 要配慮個人情報（病歴、障害情報など）を含むため、取り扱いには十分ご注意ください。

##### 音声ファイルおよび文字起こしテキスト

- 音声ファイルおよび文字起こしテキストには、生の会話内容として個人情報や機微情報が含まれる可能性が高いため、取り扱いには特にご注意ください。

##### 外部APIへのデータ送信

本システムは、文字起こしと要約の処理に外部APIを利用します。

- **文字起こし**：音声ファイルをOpenAI Whisper APIへ送信します。
- **要約**：文字起こしテキストをClaude APIへ送信します。
- これらのデータは各APIプロバイダーの利用規約に従い処理されます。
- 個人情報保護の観点から、十分ご理解のうえご利用ください。
- **外部へのデータ送信に関して、クライアントから同意を取得することをお勧めします。**

### 11-2. セキュリティ機能

本システムは、不正アクセスや情報漏洩を防ぐため、以下のセキュリティ機能を備えています。

#### 11-2-1. パスワード管理

##### パスワードポリシー
- **最小文字数**：8文字以上
- **必須**：英大文字・小文字・数字・記号をそれぞれ1文字以上含む
- **禁止**：よく使われるパスワード（password、Admin123! 等）は使用不可
- **初期パスワード**：管理者が設定し、各トレーナーは初回ログイン時にパスワード変更が必要

##### パスワードの保存
- データベース内ではパスワードを暗号化して保存します。
- システム開発事業者を含め、パスワードの平文を閲覧できません。

#### 11-2-2. アカウントの管理

##### 無効化

退職・休職などにより業務を離れたトレーナーのログインを停止します。

- **操作**：管理者がトレーナー管理画面から無効化できます。
- **動作**：無効化されたアカウントはログイン不可となります。主担当の割り当てやトレーニング記録の担当者履歴はそのまま維持されます。
- **解除方法**：管理者がトレーナー管理画面から有効化できます。
- **対象外**：
  - システム管理者アカウント
  - ログイン中の自分自身
  - 最後の有効な管理者（管理者が1名のみの場合）

##### ロック

不正アクセスや長期未使用アカウントの悪用を防ぐため、以下の条件でアカウントをロックします。

- **条件**：
  - **長期間未ログイン**：30日間ログインがない場合
  - **未使用**：アカウント発行後、一度もログインがないまま7日が経過した場合
  - **連続ログイン失敗**：5回連続でログインに失敗した場合
- **動作**：ロックされたアカウントはログイン不可となります。主担当の割り当てやトレーニング記録の担当者履歴はそのまま維持されます。
- **解除方法**：管理者がトレーナー管理画面から解除できます。
- **対象外**：
  - システム管理者アカウント

#### 11-2-3. 自動ログアウト

本システムでは、第三者の覗き見やなりすましを防ぐため、以下の2つの条件で自動的にログアウトします。

##### 時間経過によるログアウト
- ログイン状態の有効期限は120分です。有効期限を過ぎると自動的にログアウトされます。

##### 無操作によるログアウト
- 一定時間操作がない場合、ログイン状態の有効期限内であってもログアウトされます。
- ログアウトまでの時間を設定できます（推奨設定は15分。詳細は6-10-1参照）。
