# テスト仕様書（画面テスト）: カウンセリング記録管理システム

## 1. 本書について

### 1-1. 目的

本書は、画面設計書（screen-design.md）に記載された全画面について、手動テストのテストケースを定義する。各画面が仕様どおりに動作し、入力チェック・権限制御が正しく機能することを確認する。

### 1-2. テスト範囲

- ブラウザ（Windows + Google Chrome）からの手動操作によるテストを対象とする。
- 画面設計書の3章「画面詳細」に列挙された全画面を対象とする。
- DB制約・ENUM等の単体確認は、本書では画面操作の異常系として確認する（PHPUnit等の自動テストは範囲外）。
